Contenedores

Se puede tomar la analogía de un contenedor como se usa en el transporte para poder empaquetar de forma aislada distintas cargas

En el desarrollo de software sucede algo similar los contenedores son procesos que se ejecutan de forma aislada dentro de nuestro sistema operativo estos contenedores agrupan el código de una aplicación junto a sus bibliotecas y archivos de configuración o cualquier otro necesario para su correcto funcionamiento

Algunas de las ventajas de la conteirenazacion es que es que puede ejecutarse en cualquier lugar que soporte la tecnología que utilices para gestionar los contenedores

Parte de su dinamismo se debe a que el SO Host carga todos los elementos necesarios como lib,drivers, archivos de sistema caso contrario que sucede con las VM las primeros rastros de contenedores surgieron hace décadas en versiones como FreeBSD Jails y Workload Partitions pero el disruptor de la tecnología fue Docker por allá del 2013
